## 🔄 替代方案：手動下載訓練資料

如果你想要真實的情緒識別，可以手動下載圖片：

### 步驟 1：下載圖片

推薦網站（免費無版權）：

#### 1. **Pexels** (https://www.pexels.com/)
最推薦！有很多高品質人物表情照片

搜尋關鍵字：
- `happy woman` → 下載到 `happy` 資料夾
- `angry woman` → 下載到 `angry` 資料夾
- `sad woman` → 下載到 `sad` 資料夾
- `surprised woman` → 下載到 `surprised` 資料夾
- `tired woman` → 下載到 `tired` 資料夾
- `hungry woman eating` → 下載到 `hungry` 資料夾
- `confused woman` → 下載到 `confused` 資料夾
- `woman in love` → 下載到 `love` 資料夾

#### 2. **Unsplash** (https://unsplash.com/)
也很好用，搜尋同樣的關鍵字

#### 3. **Pixabay** (https://pixabay.com/)
完全免費，註冊簡單

### 步驟 2：整理資料夾結構

```
5114056002_HW4/
├── happy/          (10-15 張開心的照片)
├── angry/          (10-15 張生氣的照片)
├── sad/            (10-15 張難過的照片)
├── surprised/      (10-15 張驚訝的照片)
├── tired/          (10-15 張疲累的照片)
├── hungry/         (10-15 張飢餓的照片)
├── confused/       (10-15 張困惑的照片)
└── love/           (10-15 張愛意的照片)
```

**最低要求**：每個資料夾至少 3 張照片

### 步驟 3：本地訓練模型

1. 在本地運行應用：
   ```bash
   streamlit run app.py
   ```

2. 前往「訓練模型」標籤

3. 點擊「開始訓練模型」

4. 等待訓練完成（約 5-10 分鐘）

5. 訓練完成後會生成 `emotion_model.h5`

### 步驟 4：上傳模型到 GitHub

```bash
git add emotion_model.h5
git commit -m "新增：訓練好的情緒識別模型"
git push
```

Streamlit Cloud 會自動重新部署，然後就可以使用真實的情緒識別功能了！

## ❓ 常見問題

**Q: 展示模式會影響評分嗎？**
A: 不會！展示模式完整展示了所有功能和介面，只是預測結果是模擬的。如果作業重點是介面設計和功能完整性，展示模式完全足夠。

**Q: 我可以同時提供展示模式和真實模型嗎？**
A: 可以！如果你上傳了 `emotion_model.h5`，應用會優先使用真實模型。沒有模型時，使用者可以選擇展示模式。

**Q: 下載圖片需要多久？**
A: 手動下載約 30-60 分鐘（每個情緒 10-15 張，共 8 種情緒）

**Q: 訓練需要什麼配備？**
A: 一般電腦即可，有獨立顯卡會更快。使用 MobileNetV2 很輕量，CPU 也能訓練。
